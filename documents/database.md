# Database Schema & Security Policies

This document describes the Supabase PostgreSQL database schema, customized extensions, functions, triggers, and Row Level Security (RLS) policies implemented for **Spotme**.

---

## 1. Schema Extensions & Custom Types

### Extensions
* **`vector`**: Used to support high-performance multidimensional vector search operations. It enables storing and comparing face embeddings extracted by the AI models.

### Custom Enum Types
* **`public.event_type`**: `'marriage'`, `'hackathon'`, `'meetup'`, `'corporate'`, `'other'`
* **`public.event_status`**: `'draft'`, `'active'`, `'archived'`
* **`public.selfie_status`**: `'uploaded'`, `'processing'`, `'matched'`, `'no_face'`

---

## 2. Table Definitions

### A. `public.profiles`
Stores details of registered users (photographers and administrators). It mirrors `auth.users` via a database trigger.
* `id` (`uuid`, Primary Key): References `auth.users(id)` (On Delete Cascade).
* `full_name` (`text`): Full name of the user.
* `avatar_url` (`text`): Link to profile photo.
* `role` (`text`): Role configuration. Can be `'admin'` or `'photographer'`. Defaults to `'photographer'`.
* `phone` (`text`): Contact number.
* `bio` (`text`): Professional details.
* `plan` (`text`): Subscription level: `'free'`, `'pro'`, `'unlimited'`. Defaults to `'free'`.
* `max_events` (`integer`): Number of events allowed based on plan. Defaults to `1`.
* `max_storage_gb` (`integer`): Storage space limit in GB. Defaults to `10`.
* `created_at` / `updated_at` (`timestamptz`): Audit timestamps.

### B. `public.events`
Manages event projects created by photographers.
* `id` (`uuid`, Primary Key): Default random UUID.
* `owner_id` (`uuid`, Foreign Key): References `public.profiles(id)` (On Delete Cascade).
* `name` (`text`): Event name.
* `venue` (`text`): Location venue.
* `event_date` (`date`): Date of the event.
* `event_type` (`event_type`): Category of the event. Defaults to `'other'`.
* `cover_url` (`text`): Event banner image URL.
* `admin_name` / `admin_phone` / `admin_email` (`text`): Photographer event contact info.
* `qr_active` (`boolean`): Determines if the QR scanner onboarding is enabled. Defaults to `true`.
* `privacy_mode` (`boolean`): If `true`, hides the general gallery; guests can only view photos matched directly to their selfies. Defaults to `false`.
* `status` (`event_status`): State of the event. Defaults to `'active'`.
* `created_at` / `updated_at` (`timestamptz`): Audit timestamps.

### C. `public.event_photos`
Stores metadata of photos uploaded by the photographer.
* `id` (`uuid`, Primary Key): Default random UUID.
* `event_id` (`uuid`, Foreign Key): References `public.events(id)` (On Delete Cascade).
* `storage_path` (`text`): Path inside the `event-photos` bucket.
* `public_url` (`text`): Public access URL.
* `thumb_url` (`text`): Optimized grid thumbnail URL.
* `medium_url` (`text`): Intermediate sized lightbox URL.
* `blur_hash` (`text`): Base64 blur placeholder hash for progressive load states.
* `original_filename` (`text`): Uploaded filename.
* `file_size_bytes` (`bigint`): Size on disk.
* `mime_type` (`text`): Image file type.
* `face_indexed` (`boolean`): Flag indicating if the AI has processed this photo. Defaults to `false`.
* `face_indexed_at` (`timestamptz`): Timestamp of processing.
* `uploaded_at` (`timestamptz`): Upload timestamp.

### D. `public.guests`
Identifies guests who scanned the event QR code.
* `id` (`uuid`, Primary Key): Default random UUID.
* `event_id` (`uuid`, Foreign Key): References `public.events(id)` (On Delete Cascade).
* `phone` (`text`): Guest's WhatsApp phone number.
* `display_name` (`text`): Display name.
* `created_at` (`timestamptz`): Time of registration.

### E. `public.guest_selfies`
Tracks selfies uploaded by guests to initiate AI matching.
* `id` (`uuid`, Primary Key): Default random UUID.
* `guest_id` (`uuid`, Foreign Key): References `public.guests(id)` (On Delete Cascade).
* `event_id` (`uuid`, Foreign Key): References `public.events(id)` (On Delete Cascade).
* `storage_path` (`text`): Path inside the `guest-selfies` bucket.
* `public_url` (`text`): Selfie URL.
* `status` (`selfie_status`): Processing state. Defaults to `'uploaded'`.
* `uploaded_at` (`timestamptz`): Time of upload.

### F. `public.photo_matches`
Caches the relationship between guest selfies and event photos.
* `id` (`uuid`, Primary Key): Default random UUID.
* `event_photo_id` / `photo_id` (`uuid`, Foreign Keys): Reference `public.event_photos(id)` (On Delete Cascade).
* `guest_selfie_id` (`uuid`, Foreign Key): References `public.guest_selfies(id)` (On Delete Cascade).
* `guest_id` (`uuid`, Foreign Key): References `public.guests(id)` (On Delete Cascade).
* `event_id` (`uuid`, Foreign Key): References `public.events(id)` (On Delete Cascade).
* `similarity` (`float`): Similarity score generated by AI model (ArcFace cosine similarity). Defaults to `0.0`.
* `matched_at` (`timestamptz`): Time match was found.

### G. `public.face_embeddings`
Stores facial bounding boxes and vector descriptors extracted from event photos.
* `id` (`uuid`, Primary Key): Default random UUID.
* `photo_id` (`uuid`, Foreign Key): References `public.event_photos(id)` (On Delete Cascade).
* `event_id` (`uuid`, Foreign Key): References `public.events(id)` (On Delete Cascade).
* `embedding` (`vector(512)`): ArcFace 512-dimension vector embedding of the face.
* `bounding_box` (`jsonb`): Bounding box coordinates `{x, y, w, h}`.
* `created_at` (`timestamptz`): Insertion timestamp.

### H. `public.inquiries`
Stores inquiries received via the landing page form.
* `id` (`uuid`, Primary Key): Default random UUID.
* `name` (`text`): Submitter name.
* `email` (`text`): Submitter email.
* `phone` (`text`): Contact phone.
* `event_date` (`text`): Desired date.
* `location` (`text`): Event location.
* `event_type` (`text`): Event type.
* `guest_count` (`text`): Estimated attendance.
* `story` (`text`): Description of their event needs.
* `created_at` (`timestamptz`): Submission timestamp.

## 3. Database Indexes

* **Vector Search Index**:
  * `face_embeddings_cosine_idx` on `public.face_embeddings(embedding)` using **HNSW** (`vector_cosine_ops`). Critical for sub-second similarity matching.
* **Unique Constraints**:
  * `guests_event_phone_idx` (Unique on `event_id, phone`) - Restricts guests to registering once per event.
  * `photo_matches_guest_id_photo_id_key` (Unique on `guest_id, photo_id`) - Prevents duplicate match entries in the cache.
* **Foreign Key Performance Indexes**:
  * `events_owner_id_idx`
  * `event_photos_event_id_idx`
  * `guests_event_id_idx`
  * `guest_selfies_guest_id_idx` / `guest_selfies_event_id_idx`
  * `photo_matches_guest_id_idx` / `photo_matches_event_photo_id_idx`
  * `photo_matches_guest_event_idx` (Index on `guest_id, event_id`) - Speeds up retrieving matches on page load.
  * `event_photos_face_indexed_idx` (Partial index on `event_id` where `face_indexed = false`) - Helps background scripts find pending tasks.

## 4. Functions & Triggers

### `public.handle_new_user()`
* **Purpose**: Auto-creates a record in `public.profiles` when a new user signs up in `auth.users`.
* **Execution**: Triggered `AFTER INSERT` on `auth.users`.
* **SQL Logic**:
  ```sql
  create or replace function public.handle_new_user()
  returns trigger
  language plpgsql
  security definer
  set search_path = ''
  as $$
  begin
    insert into public.profiles (id, full_name, avatar_url)
    values (
      new.id,
      new.raw_user_meta_data ->> 'full_name',
      new.raw_user_meta_data ->> 'avatar_url'
    );
    return new;
  end;
  $$;
  ```

## 5. Row Level Security (RLS) Policies

RLS is activated on all public tables to enforce privacy barriers.

### Profiles
* **Select**: Authenticated users can only read their own profile (`auth.uid() = id`).
* **Update**: Authenticated users can only edit their own profile (`auth.uid() = id`).

### Events
* **Manage (All)**: Authenticated users can manage events they own (`auth.uid() = owner_id`).
* **Select**: Anonymous and authenticated users can view an event only if `status = 'active'` and `qr_active = true`.

### Event Photos
* **Manage (All)**: Authenticated users can manage photos only for events they own.
* **Select**: Anonymous and authenticated users can view photos from an event only if the event is active (`status = 'active'` and `qr_active = true`).

### Guests
* **Insert**: Anyone (anonymous or authenticated) can register themselves as a guest.
* **Select (Guests)**: Guests can read guest records (used for verification lookup).
* **Select (Owners)**: Authenticated photographers can view all guest records for events they own.

### Guest Selfies
* **Insert**: Anyone can upload a selfie to trigger processing.
* **Select (Guests)**: Guests can view their own selfies.
* **Select (Owners)**: Authenticated photographers can view selfies uploaded to events they own.

### Photo Matches
* **Manage (All)**: Authenticated photographers can manage matches for events they own.
* **Select**: Guests can view matches linked to their registration.

### Face Embeddings
* **Manage (All)**: Authenticated photographers can manage face embeddings for events they own.
* **Select**: Anyone can view face embeddings if the event is active.

### Inquiries
* **Insert**: Anyone can submit an inquiry from the landing page.
* **Select**: Only authenticated users with the role `'admin'` in their profile can view inquiries.

## 6. Storage Buckets & Policies

Three buckets are used to organize uploaded files:

### A. `event-covers` (Public)
* **Path**: `covers/{event_id}/{filename}.jpg`
* **Select**: Public read for anyone.
* **Insert**: Authenticated users can upload.
* **Update/Delete**: Authenticated owners can update/delete cover files starting with their `auth.uid()`.

### B. `event-photos` (Public)
* **Path**: `photos/{event_id}/{filename}.jpg`
* **Select**: Public read for anyone (filtered in the client by event status).
* **Insert/Delete**: Authenticated users can upload/delete.

### C. `guest-selfies` (Public)
* **Path**: `selfies/{event_id}/{guest_id}/{timestamp}.jpg`
* **Select**: Public read for anyone.
* **Insert**: Anyone (anonymous or authenticated) can upload a selfie.
